1. Block-level elements start on a new line and uses the entire space of its parent element(container) in a web page and disrupt the natural flow of text. Most of the common HTML elements are block-level elements including headings, paragraphs, divs, unordered and ordered lists, and tables.

2. The way block-level elements are used is entirely semantic. Meaning that there are a ton of html elements that do the same thing but are used differently based on how readable you want your html to be. The div and section elements achieve the same thing except div is more vague and section is more specific.

3. Inline elements display inline with text and can start anywhere in a line without disrupting the flow of the text. Some inline elements include links, images, bold, and span.

4. Inline elements may only contain data and other inline elements. Thus, inline elements are usually nested inside block-level elements because block-level elements may contain inline elements and other block-level elements.
